Architecture

Every layer is decentralized

Discovery

Hyperswarm DHT

Fans find each other by match name — no signalling server. Deterministic topic hash, race-free room creation.

Shared State

Autobase

Multi-writer, eventually-consistent room state. Chat, reactions, match events — merged deterministically across all peers.

Clip Sync

Hypercore

Append-only binary feeds replicate highlight clips peer-to-peer. Byte-identical, on-demand.

Local AI

QVAC Commentator

Llama 3.2 1B via @qvac/sdk. Streams live commentary from match events — runs on-device, never phones home.

🔗 Corestore ⚡ WebSocket fan-out 🌐 Web UI (vanilla JS) 🖥️ Node.js runtime

QVAC — your private commentator

A real LLM
in your pocket

★ FanMesh QVAC

Llama 3.2 1B Instruct runs locally via @qvac/sdk (llama.cpp). Loads in ~2s. Generates live match calls in English, Español, and more — streamed token-by-token. No API key. No data leaves the device.

✕ Cloud AI commentators

Depend on a server, an API key, and a stable connection. Latency, rate limits, censorship, and the single biggest failure point during peak traffic.

Censor the network? Fans still talk. Kill the cloud? The AI still commentates.

Why FanMesh wins

The fan network
that can't be shut down

◆ Censorship-resistant

No central server to block or overload. The Pears DHT means fans connect directly — governments and ISPs can't kill the conversation.

◆ Private by design

The AI commentator never sends data anywhere. Your commentary, your language, your device. Sovereign and local.

✕ Discord / WhatsApp watch parties

Central servers, cloud AI, rate limits, and a single point of failure. Gone when traffic spikes or when blocked.

✕ Plain WebRTC apps

Need STUN/TURN signalling servers. Not truly serverless. FanMesh uses the full Pears Stack — no signalling, no relay.
